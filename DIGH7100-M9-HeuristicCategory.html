Definition Category
Consistency across all aspects of the product: methods of navigation, messages and actions, meaning of buttons, and terms and icons. Congruence with known screen design principles for colour and screen layout. Consistency with ISO usability guidelines. Consistency and standards
Users understand what the system is doing and what they can do with the product from the system messages, information, and displays. Visibility of system state
The technology matches the way users think and do work, uses appropriate information flow, has typical options that users need, and includes expected actions by the system. Match between system and world
No superfluous information. System and screen design targeted to primary information users' needs. Use of progressive disclosure to display details of a category of information only when needed. The exception can be designs for expert users where screen density is preferred. Minimalist
Minimizing the amount of information and tasks users have to memorize to adequately use the technology. Product makes use of sample formats for data input, such as a calendar for date format. Minimize memory load
The technology provides prompt and useful feedback about users' interactions and actions (e.g., feedback that orders were placed). Informative feedback
The ability to tailor and customize to suit individuals' needs. Includes novice and expert capabilities (e.g., string searches). Flexibility and efficiency
Tell users what error occurred and how users can recover from the error. Not abstract or general such as "Forbidden!" Need to be precise and polite and not blame the user. Good error messages
Catastrophic errors must be prevented (e.g., mixing pediatric medication order dosing between kilograms and pounds or delivering a radiation dose with the device left wide open instead of being tailored to tumor size). Prevent errors
Users should know when a task is completed and all information is accepted. Displays should include progress toward 100% completion versus using a series of bars. Clear closure
Whenever possible, actions and interactions should be able to be undone within legal limits in electronic health records. If actions cannot be reversed, there is a consistent procedure for documenting the correction of any misinformation in the system Reversible actions
The technology uses language and terms the targeted users can comprehend and expect. Health terms are used appropriately. Use the users' language
Users initiate actions versus having the perception that the technology is in control. Avoid surprising actions, ending up in unexpected places, and loud sounds with errors. Users in control
Provide help for users within the context the actions occur (context sensitive). Embed help functions throughout the application. Help and documentation
